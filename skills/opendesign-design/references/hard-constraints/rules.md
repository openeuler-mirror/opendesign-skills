# 组件组合规则 · 硬约束

> ⚠️ 本文档定义 OpenEuler 设计稿中**组件排布、嵌套与组合**的合法规则。AI 在生成或修改 Pixso 设计稿时**必须严格遵守**，违反任何一条均视为非法布局，必须自动修正或抛出规范提醒。

---

## 1. 通用规则

### 1.1 Button（按钮）

| 规则 | 说明 |
|---|---|
| **对齐** | 按钮无论置于容器内还是页面背景中，**一律左对齐** |
| **排列优先级** | 多个按钮共存时，从左到右顺序为：`solid`（强调）→ `outline`（普通）→ `text`（文本） |
| **Solid 数量** | 同一组按钮中，`solid` 变体**有且只有一个** |
| **按钮间距** | 同组按钮之间的水平间距**固定 24px** |
| **与其他内容的上下间距** | 按钮（组）与周围其他内容或组件的上下间距**固定 24px** |

### 1.2 组件与栅格

- ✅ **所有组件必须挂靠栅格容器**，严格对齐内容区的栅格列
- ❌ **禁止组件游离在页面空白区域独立摆放**（即不得出现脱离栅格的浮动元素）

### 1.3 Card（卡片）

- ✅ 卡片与卡片之间的间距**固定使用 32px**（与硬约束 §4.1 水槽宽度一致）
- ❌ 禁止使用 16 / 20 / 24 / 40px 等非标准卡片间距

---

## 2. 禁止组合黑名单（Skill 自动校验拦截）

> 下列组合一经发现，Skill 自动拦截并提示修正，**不得静默放行**。

| # | 禁止行为 |
|---|---|
| 1 | ❌ **L-Banner 塞进卡片、弹窗、子内容区块**（L-Banner 仅允许出现在页面顶级区块中） |
| 2 | ❌ **同一区块内 L + M + S 不同尺寸 Banner 并排** |
| 3 | ❌ **按钮放在标题 / 富文本上方**（按钮必须位于对应内容的下方） |
| 4 | ❌ **表单按钮分散在各表单项中间**（表单按钮统一置于表单末尾） |
| 5 | ❌ **卡片内部横向混放多种不同规格组件** |

---

## 3. 页面生成时 AI Skill 自动校验项

> 生成或修改设计稿时，Skill 必须在产出前完成以下校验，不通过则自动修正或抛出规范提醒：

```
□ 1. Banner 尺寸与所在页面 / 容器是否匹配（不允许 L-Banner 嵌入子区块）？
□ 2. 同区块内 Banner 尺寸是否统一（不允许多尺寸并排）？
□ 3. 按钮排列顺序：solid → outline → text，且 solid 有且只有一个？
□ 4. 按钮对齐方式是否为左对齐？
□ 5. 按钮间距及按钮与周围内容上下间距是否均为 24px？
□ 6. 所有组件是否挂靠栅格容器（无游离元素）？
□ 7. 卡片间距是否统一为 32px？
□ 8. 组件排布顺序、嵌套层级是否合法（无黑名单组合）？
```

任一项不通过 → **自动修正到合规状态**，无法自动修正时停下询问用户，**禁止静默放行**。
