# 变量映射规则 · 硬约束

> ⚠️ 本文档定义 OpenEuler 设计稿中**所有**字号、行高、间距、栅格、图标尺寸的合法取值。AI 在生成或修改 Pixso 设计稿时**必须严格选用本表中的值**，禁止四舍五入、推断、造数，或使用"看起来合理"的非清单数值。任何不在本表内的几何数值都视为非法。

---

## 1. 断点定义

| 设备 | 屏宽范围 | 使用列 |
|---|---|---|
| **MB（移动端）** | 360 ~ 600px | 下方所有表格的 **MB** 列 |
| **PC（桌面端）** | > 1680px | 下方所有表格的 **PC** 列 |

**核心原则**：

- 单个设备类别内（如 MB 360→600px），**字号、间距均不做变化**，只做横向布局拉伸（以 MB 端为主进行设计）
- 跨设备类别（PC ↔ MB）切换时，按下方各表执行 1:1 映射

---

## 2. 文字（字号 / 行高）

> 🔴 **本节最高优先级铁律**：生成**任何**可编辑的 Pixso 设计稿时，**全局字体一律使用「鸿蒙黑体（HarmonyOS Sans / HarmonyHeiTi）」**，无任何例外。禁止使用 PingFang、苹方、思源、微软雅黑、Inter、Roboto、系统默认字体或任何其他字族。设计稿中每一个文字图层的 `font_family` 必须显式设为鸿蒙黑体。

下表中"角色"指文字在页面信息层级中的语义身份。**字号与行高在同一行内成对锁定，禁止拆分搭配**。

| 角色 | PC（字号 / 行高） | MB（字号 / 行高） | MB 上的角色重命名 |
|---|---|---|---|
| 一级数据展示 | 56px / 80 | 22px / 30 | — |
| 二级数据展示 | 48px / 64 | 20px / 28 | — |
| 三级数据展示（楼层标题） | 40px / 56 | 18px / 26 | — |
| 一级标题 | 32px / 44 | 18px / 26 | — |
| 二级标题 | 24px / 32 | 16px / 24 | — |
| 三级标题（列表卡片 < 3 列） | 22px / 30 | 16px / 24 | — |
| 四级标题（列表卡片 = 4 列） | 20px / 28 | 16px / 24 | — |
| 大号正文 | 18px / 26 | 14px / 22 | — |
| 常规正文 / 标题辅助信息 | 16px / 24（段高 12px） | 14px / 22 | 常规正文 |
| 提示文本 1 | 14px / 22 | 12px / 18 | 标题辅助信息 |
| 提示文本 2 | 12px / 18 | 10px / 16 | — |

**约束**：

- ✅ PC 合法字号集合：`{56, 48, 40, 32, 24, 22, 20, 18, 16, 14, 12}`
- ✅ MB 合法字号集合：`{22, 20, 18, 16, 14, 12, 10}`
- ✅ 字号与行高**必须同行配对**：PC 32px 行高只能是 44；MB 16px 行高只能是 24
- ❌ 禁止使用 13 / 15 / 17 / 19 / 21 / 23 / 26 / 28 / 30 / 36 等中间字号
- ❌ 禁止跨设备混用：MB 设计稿出现 32px/44 行高 = 错

## 3. 间距

| Token | PC | MB |
|---|---|---|
| 间距 1 | 4px | 4px |
| 间距 2 | 8px | 8px |
| 间距 3 | 12px | 8px |
| 间距 4 | 16px | 8px |
| 间距 5（一般用于组件与组件之间的**左右**间距） | 24px | 12px |
| 间距 6（一般用于组件与组件之间的**上下**间距） | 32px | 12px |
| 间距 7（楼层标题到楼层内容间距） | 40px | 12px |
| 间距 8 | 48px | 16px |
| 间距 9 | 64px | 24px |
| 间距 10（楼层之间的距离） | 72px | 32px |

**约束**：

- ✅ 所有 `padding` / `margin` / `gap` 必须从对应设备列中选择
  - PC 合法集合：`{4, 8, 12, 16, 24, 32, 40, 48, 64, 72}`
  - MB 合法集合：`{4, 8, 12, 16, 24, 32}`
- ✅ 楼层标题到内容的间距**固定**使用**间距 7**（PC 40px / MB 12px）
- ✅ 楼层之间的距离统一使用**间距 10**（PC 72px / MB 32px）
- ❌ 禁止 6 / 10 / 14 / 18 / 20 / 28 / 36 / 56 等未列出数值
- ❌ MB 端禁止出现 40 / 48 / 64 / 72px 等 PC 专属数值

## 4. 栅格

### 4.1 实际布局速查（生成页面时使用）

> 🔴 **生成 Pixso 设计稿时，画布尺寸与内容区位置必须严格按下表落实**。这是 AI 真正动手时的取值表。底层栅格结构定义见 §4.2（仅参考，不直接用于布局）。

| 项 | PC | MB |
|---|---|---|
| 画布总宽度 | 1920px | 360px |
| 内容区宽度 | 1488px | 312px |
| 内容区距画布两边边距 | 216px | 24px |
| 栅格列数 | 24 列 | 4 列 |
| 水槽宽度（card 与 card 的上下左右间距） | 32px | 12px |

**校验**：内容区宽度 + 两边边距 × 2 = 画布总宽度
- PC：1488 + 216 × 2 = 1920 ✓
- MB：312 + 24 × 2 = 360 ✓

### 4.2 设计系统栅格 spec（底层定义，参考用）

> ℹ️ 下方 JSON 为设计系统底层栅格结构定义，描述**画布栅格基础参数**。**`margin` 字段 ≠ §4.1 中"内容区距画布两边边距"**（前者是栅格安全间距 PC 64 / MB 24，后者是实际内容布局边距 PC 216 / MB 24）。
>
> 落地页面时**以 §4.1 为准**，本节仅作为底层 spec 参考。`vp` 为 HarmonyOS 密度无关单位，与 `px` 1:1 对待。

```json
"grid": {
  "pc": {
    "breakpoint": "1920px",
    "columns": 24,
    "gutter": "32vp",
    "margin": "64px",
    "width": "1920px"
  },
  "mb": {
    "breakpoint": "360px",
    "columns": 4,
    "gutter": "12vp",
    "margin": "24px",
    "width": "360px"
  }
}
```

字段含义：

| 字段 | 含义 |
|---|---|
| `breakpoint` | 设计稿对齐的基准断点（PC 1920 / MB 360），即设计稿应以此宽度为基础进行布局 |
| `width` | 画布固定宽度（生成 Pixso 设计稿时的 frame 宽度） |
| `columns` | 栅格列数（PC 24 / MB 4） |
| `gutter` | 列间距 / 水槽宽度（PC 32 / MB 12） |
| `margin` | 栅格安全间距（PC 64 / MB 24）。**注：此值 ≠ §4.1 实际内容布局边距（PC 216 / MB 24），生成页面时以 §4.1 为准** |

**约束**：

- ✅ PC **必须**使用 24 栅格，MB **必须**使用 4 栅格
- ✅ PC 整页画布宽度**固定 1920px**，MB 整页画布宽度**固定 360px**
- ✅ PC 实际内容区**宽度固定 1488px、距画布两边各 216px**；MB 实际内容区**宽度固定 312px、距画布两边各 24px**（见 §4.1）
- ✅ 卡片之间间距严格使用 PC 32px / MB 12px
- ❌ 禁止使用 6 / 8 / 12 / 16 / 20 等其他栅格数
- ❌ 禁止使用 1440 / 1680 / 375 / 414 等非基准画布宽度
- ❌ MB 端不要把卡片间距设成 16 / 24 / 32px

## 6. AI 自检清单（每次生成 / 修改前必走）

```
□ 1. 当前目标设备：[ ] MB   [ ] PC   → 已锁定对应列
□ 2. 所有字号是否来自该设备列中的字号集合？
□ 3. 字号与行高是否成对（未拆分搭配）？
□ 4. 所有 padding / margin / gap 是否来自该设备列中的间距集合？
□ 5. 栅格列数 / 水槽 / 边距是否符合该设备列？
□ 6. 图标是否正方形，且尺寸来自该设备列的图标集合？
□ 7. 是否存在"看起来合理但未列出"的数值？
□ 8. 【落地页】页面结构是否符合：导航 → Banner → 楼层(s) → 页脚？
□ 9. 【落地页】楼层间距是否统一使用间距 10（PC 72px / MB 32px）？
```

任一项不通过 → **立即回滚到清单值**，禁止"差不多"。无法对应到清单时，停下询问用户，**不要自行造数**。

## 7. 不在本表内的属性

下列属性**不受**本变量映射规则约束，按各自来源处理：

| 属性 | 来源 |
|---|---|
| 色值（`color-*`） | 远端 Token JSON，见 [SKILL.md 数据资源](../../SKILL.md#数据资源) |
| 字重（`font_weight-*`） | 远端 Token JSON |
| 阴影（`shadow-*`） | 远端 Token JSON |
| 圆角 | 见 [references/components/](../components/) 各组件文档（按钮 4 / 16 / 100px、输入框/卡片/Tag 4px 等） |
| 按钮 / 输入框 / Tag 等**组件级**几何参数 | 见 [references/components/](../components/) 各组件文档 |
| 描边宽度 | 全局 1px INSIDE，不随设备变化 |

## 8. 落地页模板规则（page-building）

> ⚠️ 本节定义**标准落地页**的页面结构与区块顺序约束。AI 在生成或补全落地页设计稿时，**必须**严格遵循下方结构，不得随意增删必选区块或调整顺序。

### 8.1 标准结构（从上到下）

| 顺序 | 区块 | 必选 / 可选 | 说明 |
|---|---|---|---|
| 1 | **顶部导航** | 必选 | 页面第一个内容区块，常驻吸顶 |
| 2 | **Banner** | 可选 | 若存在，必须紧贴导航，位于页面第二个区块 |
| 3 | **楼层（可多个）** | 必选 | 详见 §8.2；楼层间距遵循间距 10 |
| 4 | **页脚** | 必选 | 常驻页面最底部 |

### 8.2 楼层内部结构

每个完整楼层由以下子结构组成：

```
楼层
├── 楼层标题              （必选）
├── 楼层标题描述性文本    （可选）
└── 楼层内容
    ├── 操作 Tab          （可选）
    ├── 卡片布局 / 表格等容器  （内容主体）
    └── 操作按钮          （可选）
```

### 8.3 约束

- ✅ **顶部导航**：必须是页面第一个内容区块，且**常驻吸顶**（`position: sticky / fixed`）
- ✅ **Banner**：若存在，必须是页面第二个内容区块，**紧贴导航**，不得有其他区块插入其间
- ✅ **楼层数量**：一个页面可包含**多个**楼层，数量不限
- ✅ **楼层间距**：相邻楼层之间统一使用**间距 10**（PC **72px** / MB **32px**），与 §3 保持一致
- ✅ **页脚**：每个页面必须有页脚，**常驻页面最底部**，不得被其他内容覆盖
- ❌ 禁止在导航之前放置任何内容区块
- ❌ 禁止在导航与 Banner 之间插入任何其他区块（若 Banner 存在）
- ❌ 禁止省略顶部导航或页脚
- ❌ 禁止楼层之间使用非间距 10 的数值（如 PC 64px / MB 24px 等）

---

## 9. 突破规则的唯一通道

如确需使用非清单值（如品牌特殊页面、营销 banner 等），**必须**在回复中显式声明：

> ⚠️ 此处使用了非标准值 `XXpx`，不在变量映射白名单内。原因：[说明]。是否确认覆盖？

由用户拍板后才能落到设计稿中。**默认行为永远是回退到最近的合法值**，而不是静默放行。
