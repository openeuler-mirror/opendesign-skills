# 组件 HTML 实现质量保证清单 (Component HTML QA Checklist)

> 📋 **文档定位**：OpenDesign 设计系统 · 组件 HTML 实现规范  
> 🔗 **适用范围**：所有从 Markdown 设计规范生成的交互式 HTML 文档  
> ⚡ **使用场景**：开发自检 / Code Review / QA 验收 / 新人培训  

---

## 📖 使用指南

### 如何使用本清单

```
1️⃣ 准备阶段：获取目标组件的 MD 设计规范文档
2️⃣ 提取阶段：从 MD 中提取所有设计参数（填入 §2 参数表）
3️⃣ 实现阶段：严格按照参数实现 HTML
4️⃣ 检查阶段：逐项勾选本清单（✅ 通过 / ❌ 不通过 / ⚠️ 需确认）
5️⃣ 验收阶段：交叉验证 + 浏览器测试
6️⃣ 归档阶段：保存检查记录 + 截图证据
```

### 检查结果标记说明
- ✅ **通过** - 完全符合规范
- ❌ **不通过** - 存在问题，必须修复
- ⚠️ **需确认** - 边界情况，需与设计师/产品确认
- ➖ **不适用** - 该项对当前组件无意义
- 📝 **备注** - 记录特殊处理或例外情况

## 🚫 十、常见陷阱与避坑指南

### 10.1 绝对不能犯的错误（❌ 致命级）

| # | 错误类型 | 错误示例 | 正确做法 | 后果 |
|---|---------|---------|---------|------|
| 1 | **禁用态颜色模拟** | 用 `filter: brightness(0.9)` 模拟 disabled | 使用精确的 RGB 值（如 `#84A1DC`） | 视觉不符合规范，无法通过验收 |
| 2 | **忽略官方资源** | 手写 SVG 或用 CSS 画图标 | 必须引用 `assets/` 目录下的官方文件 | 图标样式不一致，维护困难 |
| 3 | **遗漏交互状态** | 只实现了 Default 和 Selected | 必须实现全部 5 种状态（含 Disabled） | 功能不完整，用户体验差 |
| 4 | **单位错误** | 写成 `gap: 8rem` 而不是 `gap: 8px` | 严格对照 MD 的单位标注 | 布局完全错乱 |
| 5 | **Token 名拼写错误** | `color-primary1` 漏了 `--o-` 前缀 | 完整写法 `--o-color-primary1` | 样式不生效 |

### 10.2 容易出错的地方（⚠️ 警告级）

| # | 易错点 | 典型错误 | 检查方法 |
|---|-------|---------|---------|
| 1 | **间距不一致** | 有的地方 8px，有的地方 10px | 全局搜索 `gap\|margin\|padding` 统一检查 |
| 2 | **SVG 尺寸错误** | 设置为 `width="16"` 而非 `"24"` | 查看 MD 中的"SVG 文件尺寸"字段 |
| 3 | **Hover 效果过度** | 颜色变化太大或添加不必要的动画 | 保持微妙变化（brightness 1.1-1.2） |
| 4 | **禁用态可点击** | 忘记加 `pointer-events: none` | 检查 cursor 和事件绑定 |
| 5 | **移动端触摸区域太小** | 点击区域只有 16×16px | 最小 44×44px（Apple HIG 规范） |

### 10.3 最佳实践建议（💡 优化级）

| # | 建议 | 收益 | 实施难度 |
|---|------|------|---------|
| 1 | **使用 CSS 变量体系** | 主题切换方便，维护成本低 | 低 |
| 2 | **添加平滑滚动导航** | 提升文档浏览体验 | 低 |
| 3 | **实现暗黑模式切换** | 增强演示完整性 | 中 |
| 4 | **添加打印样式** | 方便离线查看 | 低 |
| 5 | **集成 Lighthouse CI** | 自动化性能监控 | 高 |

| **一、设计参数提取** | 15% | 基础中的基础，不准确则后续全错 |
| **二、实现完整性** | 15% | 内容覆盖度和准确性 |
| **三、视觉还原度** | 20% | 最直观的质量体现 |
| **四、交互行为** | 15% | 功能性和可用性 |
| **五、可访问性** | 5% | 合规性和包容性 |
| **六、响应式兼容** | 10% | 多端适配能力 |
| **七、代码质量** | 10% | 可维护性和扩展性 |
| **八、性能体验** | 5% | 加载速度和流畅度 |
| **九、测试验收** | 5% | 验证充分性 |
| **总计** | **100%** | |

### 11.2 评级标准

| 等级 | 分数范围 | 含义 | 处理方式 |
|------|---------|------|---------|
| 🏆 **S 级** | 95-100 | 完美实现，可直接发布 | 作为范例推广 |
| ✅ **A 级** | 85-94 | 优秀，微小瑕疵可接受 | 修复后发布 |
| 👍 **B 级** | 75-84 | 良好，存在少量问题 | 必须修复后再发布 |
| ⚠️ **C 级** | 60-74 | 及格但需大幅改进 | 返工重做重点模块 |
| ❌ **D 级** | < 60 | 不合格 | 全面返工 |

### 11.3 一票否决项

出现以下任一情况，**直接判定为不合格（D 级）**：

- ❌ 禁用态颜色使用 CSS filter 模拟而非精确值
- ❌ 未使用官方资源文件（手绘图标/SVG）
- ❌ 缺少超过 30% 的必需交互状态
- ❌ 存在 JavaScript 运行时报错
- ❌ 在主流浏览器中无法正常显示
- ❌ 安全漏洞（XSS、注入攻击等）
